In its original implementation XPTable provided a simple means of specifying the width of Columns via the integer property - Column.Width. It's value could be set at Design time and changed at any time programatically. This simple, but straightforward implementation, has been enhanced by augmenting Column's integer Width property with a composite (expandable) property, ColumnWidth. The ColumnWidth property is predictably implemented by the ColumnWidth class. The Column.Width property has been retained, but rather than acting on an integer backing field it now acts on the Width property of the Column's ColumnWidth property, also it is no longer a browsable property.


[System.ComponentModel.TypeConverter(typeof(XPTable.AddIns.PropertySequencer))]

public class ColumnWidth : System.ComponentModel.ISupportInitialize


Notes :

PropertySequencer is a TypeConverter that is packaged with XPTable because it is used by this class, however it does not intrinsically belong to XPTable in that it can be used on any property where one might ordinarily use the ExpandableObjectConverter, it provides control over the order in which properties are presented in the Property grid.


The ISupportInitialize interface is implemented in order that the EndInit method can cross-validate the values of the ColumnWidth properties.

Mode

ColumnWidth.modes

One of the four modes must be specified, they are Normal, Autosize, Percent and Stretch – see Nested Types for further discussion.


The availability of modes for a Column may be restricted by endowing the Column class with a ColumnWidth.constraintsAttribute. The XPTable.Column class is abstract and is implemented via “type” specific classes such as TextColumn, ButtonColumn, each of which has its own ColumnWidth.constraintsAttribute.

SingleLineHeaders

bool

The value of this property determines whether or not the width od thew HeaderText value is taken into consideration when calculating a Column's width. A Column's width is computed if the ColumnWidth.Mode property is Autosize, Percent or Stretch. If this property is true then the minimum with will be the width of the Column's HeaderText value.

NormalWidth

int

This specifies the width, expressed as a pixel count, of a Column whose ColumnWidth.Mode is ColumnWidth.modes.Normal.

MinimumWidth

int

This specifies the minimum width of a Column whose ColumnWidth.Mode is ColumnWidth.modes.Autosize or ColumnWidth.modes.Stretch. If SingleLineHeaders is true then it cannot be less that the width of the Column's Header property.

MaximumWidth

int

This specifies the maximum width of a Column whose ColumnWidth.Mode is ColumnWidth.modes.Autosize or ColumnWidth.modes.Stretch. It must not be less than ColumnWidth.MinimumWidth.

Percent

decimal

For Column's whose ColumnWidth.Mode is ColumnWidth.modes.Percent this property specifies the percentage of the Table's remaining width (i.e after the width of all other Column has been determined) that should be allocated to this this Column.

Width

int

When read from the Property provides the width, expressed as a pixel count, of the Column, in the case of Columns have a ColumnWidth.Mode of ColumnWidth.modes.Autosize or ColumnWidth.modes.Stretch or ColumnWidth.modes.Percent this is a computed value.


When written to the Property will only act upon the value if the Column's ColumnWidth.Mode is ColumnWidth.modes.Normal, in which case the ColumnWidth.NormalWidth property is set to the supplied value, in the other cases it is ignored.

AllowableMinimum

{readonly)

int

Provides the allowable minimum width, as determined by the value of SingleLineHeaders

Patch(ColumnWidth.changes.Data _patchData)

ColumnWidth

This method is necessitated by the architecture of event processing within XPTable, which is predicated on a reverse cascade (or bubble up) model.


What the method does is to create a shallow copy of the ColumnWidth object, then it apples the OldValue property of the _patchData parameter to the property specified by the _patchData.ChangeType property.


In simple terms it “reconstitutes” the ColumnWidth object to the state it was in prior to the Change event.

ColumnWidth.modes


The ColumnWidth.modes enumeration has four values as described below. It should be noted that the reason for endowing the ColumnWidth.modes Type with the Flags attribute is not for the usual reason of allowing an object of the type to have multiple values, ColumnWidth.Mode property can only have one value. Having the ColumnWidth.modes values specified as powers of 2 makes for simpler definition of the permissible modes in the ColumnAttribute by OR'ing (|) the required values together (e.g. ColumnWidth.modes.Normal | ColumnWidth.modes.Percent) rather than having to construct object array.
